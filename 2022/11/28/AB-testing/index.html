  1. What is A/B testing?
    AB testing
    An A/B test is an experiment
  • All elements are constant
  • Except for one variable
  • used in industry to make decisions

Tipically it compares a control group and a treatment group.
All variables are identical between two groups except for a factor that’s being tested.

Variants:

  • 2 variants: A/B test
  • more than 2 variants: A/B/N test

Appear frequently in Data Science Interview

  • Typically asked together with metrics problems
  • Can include any A/B testing components
    • Developing new hypothesis
    • Designing A/B tests
    • Evaluating results
    • Making decisions
  1. Why run A/B test?
    Goal: Make data driven decisions
    When results are reliable and repeatable

Make results reproducible
Change to product -> Change to metrics
Example:
Change to colors -> Change to engagement
Others stay the same

3. How long to run an A/B test?
2.1 Determine the sample size

  • Type II error or Power
  • Significance level
  • Minimum detectable effect

sample size = 16 * variance / (effect size)**2

  • More samples if variance is larger
  • Less samples if effect size is larger

How to estimate parameters?

  • sample variance is obtained from data

  • How to estimate sample difference(effect size)?

    • We don’t know before experiments
    • Use minimum detectable effect

    Minimum detecable effect

    • The smallest different matters in practice
    • Eg. 0.1% increase in recenue
    • Decided by multiple stakeholders
  1. Multiple testing problem
    Test multiple variants of a feature
  • colors
  • Homepage design

Sample questions:

  • 10 tests are running with different landing page

  • 1 case won and p value < 0.05

  • Shoud you make the change? - NO

  • Should not use the same significance level

  • Probability of false discovery increases

Eg. There are 3 groups, what is the chance of at least one false positive?
Pr(no false positive) = (1-0.05)^3 = 0.95^3
Pr(at least 1 false positive) = 1 - Pr(no false positive) = 0.143

  • Type I error over 14%

Solutions:

4.1 Bonferroni correction

  • Significance level/number of tests
    This method is too conservertive

4.2 Control False Discovery Rate(FDR) ?
FDR = E[false positives/rejections]
The expected value of number of false positives divided by number of rejections.
It measures out of all the rejections of the null hypothesis that is all the metrics
that you declare to have a statistically significant difference how many of them have
a real difference as opposed to how many were flase positives.

This only makes sense when you have a huge number of metrics.

4.3 FDR(False discovery rate) Adjustment
Benjamin – Hochberg Adjustment

  1. Rank p-values 𝑃 from low to high

  2. Find the largest k such that 𝑃 ≤ 𝐾/n ∗ 𝛼

  3. Reject experiments 1,2,..,k. Accept experiment k+1, …, n.

  4. Novelty and primacy effect

5.1 Primacy effect(change aversion)

  • People are reluctant to change

5.2 Novelty effect

  • People welcome the changes and use more

Effect will NOT last long. Repeat usage declined when effect wears off.

How to deal with them
5.3 Rule out the possibility

  • Run the test only on first time users

5.4 If test is already running

  • Compare first time users to old users in treatment group
  • If new feature has no impact on new users but only performers better in old users. That’s a sign of novelty effect.
  1. Interference between variants
    Typical design:
  • Split users randomly
  • Users are independent

Cases when assumption fails:

  • Social network: Eg. Facebook, Linkedin, Twitter
  • Two-sided markets: Eg. Uber, Lyft, Airbnb

6.1 Network effect:
Users tend to use a feature or product more ofthen if their friends use it.

  • User behaviors are impacted by others
  • The effect can spillover the control group
  • The difference underestimates the treatment effect
  • Actual effect > treatment effect

6.2 Two-sided markets:

  • Eg. Uber, Lyft, Airbnb
  • Resources are shared among control and treatment group which caused compete in control and treatment group.
  • Actual effect < treatment effect
  1. Dealing with interference
    Sample questions:
  • A new feature provides coupons to our riders
  • Goal: increase rides by decreasing price
  • Testing strategy: evaluate the effect of the new feature

7.1 Two-sided markets:
Geo-based randomization

  • Split by geolocations
  • Big variance since markets are unique

Time-based randomization

  • Split by day of week
  • Assign all users to either treatment or control
  • Only when treatment effect is in short time(Surge price)
    Does not work:
  • Treatment effect takes a long time
  • A referral program

7.2 Social network

7.2.1 Create network clusters:

  • People interact mostly whthin the cluster
  • Assign clusters randomly

7.2.2 Ego-network randomization
where a cluster s comprised of an “ego” (a focal individual), and her “alters” (the
individuals she is immediately connected to)

  • Originated from Linkedin
  • A cluster is composed of an “ego” and her “alters”
  • One-out network effect: user either has the feature or not
  • It’s simpler and more scalable

    Using Hypothesis Testing to Analyze A/B Testing results
    Pratical significant boundry: 0.01
    Significance level alpha: 0.05

Two-sample test

  1. Which hypothesis test to use?
  • Bernoulli population: either clicks or doesn’t click
  • control group: n*p = 1000 * 1.1% = 11
  • Treatment group: n*p = 1000 * 2.3% = 23
  • Test statistic follows Z-distribution

Measurements

  • Users clicked Xct, Xtr
  • Total number of users nct, ntr
    Pct = Xct/nct
    Ptr = Xtr/ntr

  1. What is the null hypothesis?
    d = ptr - pct
    Null hypothesis
    Ho: Pct = Ptr, d = 0
    d_mean ~ N(0, SE^2)

  2. Is the result statistically significant?
    Significance level alpha:0.05
    Test statistics
    TS = (Ptr - Pct)/SE
    Estimate the difference, d
    Ptr - Pct = 0.012

Estimate standard error

  • Choose a SE can represent both groups
  • “Pooled” standard error
    • “Pooled” probability of a click, p
    • Total probably across 2 groups p_pool = (Xct + Xtr)/(nct + ntr) = (11 + 23)/(1000 + 1000) = 0.017
    • S_pool = (p*(1-p)*(1/nct + 1/ntr))^0.5 = 0.00578

critical Z score with significance level = 0.05 is 1.96
Test score > 1.96 or < -1.96 which means the result is statistically significant

Checking statistical significance

  • Check if CI overlaps with 0
  • If it does, result is not statistically significant
  • Equivalent to compare TS with critical value
    If the confidence interval (with your chosen level of confidence) includes 0, that implies you think 0 is a reasonable possibility for the true value of the difference. In general, by ‘significant’ people usually mean that they no longer believe the null hypothesis (0) is a reasonable possibility. Note that if a 95% CI doesn’t include 0, the 𝑝-value would be <.05, which is the conventional cutoff for ‘significance’.
  1. Is the result practically significant?
    Pratical significance boundry:0.01

Confidence interval of d

  • center of CI = 0.012
  • Width of CI(margin of error)
    margin_error = Z * SE_pool = 1.96 * 0.00578 = 0.0113

CI of d: 0.012 +/- 0.0113 = 0.0007 ~ 0.0233
