Stack

  1. 模拟堆
    如何手写一个堆？
    堆是一个完全二叉树，父节点与子节点的关系可以用下表来联系
    左儿子：2n
    右儿子：2n + 1
    就数组模拟来说，知道数组的下标就知道结点在堆中的位置。所以核心就在于即使有down和up操作也能维护堆数组的下标（k）和结点（idx）的映射关系。比如说：h[k] = x, h数组存的是节点的值，按理来说应该h[idx]来存，但是节点位置总是在变的，因此维护k和idx的映射关系就好啦!(维护下标k和第idx个插入的数之间的映射关系)

举例： 用ph数组来表示ph[idx] = k(idx到下标), 那么结点值为h[ph[idx]], 儿子为ph[idx] * 2和ph[idx] * 2 + 1, 这样值和儿子结点不就可以通过idx联系在一起了吗？

  • 第idx个插入的数在堆中的下表是k。h[k]表示堆中下标为k堆数的值。所以这个插入的数的数值为h[ph[idx]]。左儿子的下标为ph[idx] * 2, 右儿子的下标为ph[idx] * 2 + 1

  • ph[idx] = k: 第idx个插入的数在堆中的下标为k

  • hp[k] = idx: 堆中下标为k的数是第idx个插入的数

  1. 插入一个数

    • 在堆这个数据结构中，数据的插入都是插入到堆尾，然后再up
    • heap[++ size] = x; up(size)

  2. 求当前集合中的最小值

    • root值
    • heap[1]

  3. 删除最小值

    • root与最后一个数交换
    • 删除最后一个数
    • 然后把新的root down一遍
    • heap[1] = heap[size];size –; down(1)

  4. 删除任意一个元素

    • 删除操作，三个步骤：
      1. 找到第idx个插入元素在堆数组中的位置（堆数组下标）
      2. 与堆尾元素交换
      3. 在原来第idx个元素所在的位置进行down和up操作。（up，down，swap操作的都输入都是下标）
    • heap[k] = heap[size]; size –; down[k]; up[k]; (只会执行一个)

  5. 修改任意一个数

    • heap[k] = x; down(k); up(k)

3个Functions来实现一个堆

  • h[k] = x:存储的是节点的值
  • ph[x]:第x个插入的数在堆里的下标
  • hp[x]:堆中下标是x的数是第几个插入的
  1. heap_swap(a,b)
    a,b: the index of the numbers in stack
  • swap the numbers -> heap_swap(h[a], h[b])
    • h[index_a] = num_a, h[index_b] = num_b –> h[index_a] = num_b, h[index_b] = num_a
  • swap ph[hp [a]], ph[hp [b]]. 第hpa个和第hpb个插入的数在stack中的index下标交换
    • ph[hp[index_a]] = index_a, ph[hp[index_b]] = index_b –> ph[hp[index_a]] = index_b, ph[hp[index_b]]] = index_a
  • swap the hp[a], hp[b] -> heap_swap(hp[a], hp[b]). 本来下标为a的num是idx1个插入的数，下标为b的num是idx2个插入的数。现在要交换，下标为b的num是idx1个插入的数，下标为a的num是idx2个插入的数。
    • hp[a] = idx1, hp[b] = idx2 –> hp[a] = idx2, hp[b] = idx1

理解heap_swap

  • Assume hp[index_1] = 1, hp[index_2] = 2. 原本第1个插入的数的下标是index_1, 第二个插入的数的下标是index_2. 经过交换后，第1个插入的数的下标变成index_2，第二个插入的数的下标变成index_1. 所以 ph[hp[index_1]] = ph[1] = index_2, ph[hp[index_2]] = ph[2] = index_1. 下标交换之后，现在index_1上的number是第二个插入的数字，所以hp[index_1] = 2， 同理 hp[index_1] = 1. 所有hp 也要交换。
