Window Function

  • DENSE_RANK() OVER(PARTITION BY a ORDER BY b (ASC|DESC))
  • RANK() OVER(PARTITION BY a ORDER BY b)
  • ROW_NUMBER() OVER(PARTITION BY ORDER BY )

  • FIRST_VALUE()
  • LAST_VALUE()
  • LAG() - value in the Above row
  • LEAD() - value in the below row
  • PERCENTILE_RANK()
    • x: 当前行的rank
    • (x-1)/ total count of rows
    • PERCENTILE_RANK = 0.5 -> Median

  • MIN()
  • SUM() OVER(PARTIION BY A ORDER BY) –> Cumulative sum

DATE/TIME

  • Current datetime
  • CURRENT_TIMESTAMP
  • GETDATE()

Convert datetime to date

  • CONVERT(CURRENT_TIMESTAMP, DATE)
  • CONVERT(value, type)
  • DATE_FORMAT(“2017-06-15”, “%Y-%m-%d”)

The DATEADD() function adds a time/date interval to a date and then returns the date.

  • DATEADD(year/month/week/day/hour/minute/second, 1, ‘2017/08/25’)

The DATEDIFF() function returns the difference between two dates.
end_date - start_date

  • DATEDIFF(year/month/week/day/hour/minute/second, start_date, end_date)

  • The CHAR_LENGTH() function return the length of a string (in characters).

  • REPLACE(string, from_string, new_string)
    The REPLACE() function replaces all occurrences of a substring within a string, with a new substring.

  • REVERSE(string)
    REVERSE() function reverses a string and returns the result.

We are studying ecommerce advertisers on wechat over a certain time period (say a week). The time period does not matter for this problem.

You are given 2 Tables:
adv_info: advertiser_id, ad_id, spend (primary key: ad_id)
ad_info: ad_id, user_id, price (primary key: ad_id, user_id)

Adv_info table contains information on advertisers. Advertiser_id is id of advertiser; ad_id is id of an ad being run by advertiser; spend is amount of money in $ that advertiser pays wechat for ad-id to show it to Wechat users; price is the revenue for a specific ad.
Q1: What would be the average advertiser spend on Wechat? Your query should return a single number.
Q2a: find advertisers with at least one conversion (click the ad or buy the product)
Q2b: what % of advertisers have at least one converted user.
Q3: We want to come up with an advertiser level metric that quantifies how well wechat advertising is working for advertisers. This should be based on the above 2 tables. After coming up with this metric, we want to compute it for each advertiser.
